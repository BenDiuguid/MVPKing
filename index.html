What is Meteor?

Open Source platform for building apps in JS

Built to power the next generation apps

Perfect for Hackathons! :D

One language everywhere

Isomorphic/Universal APIs mean that the same code, written in the same language, can run on both client and server.

Radically less code

Accomplish in 10 lines what would otherwise take 1000, thanks to a reactive programming model that extends all the way from the database to the user's screen.

Fullstack package system

Fullstack package system complete with one build tool that builds all parts of your app.

Live updates

Data updates live on the screen as it changes. Users can collaborate seamlessly.

Browser and Mobile

A great experience both on the web and as a phone or tablet app.

Hot deploys

Type one command to push your app into production and update all connected browsers and devices. No need to go through the App Store.

Galaxy

Containerized hosting solution by MDG

Using Docker and AWS

Handles a lot of the issues of scaling automagically

FREE app.meteor.com hosting

Bread & Butter

File/Folder Structure

  • /server - Code only run on server
  • /client - Code only run on client
  • /anythingElse - Code runs on both the server and client
  • /public - static assets like pics

Load Order

  • /lib - Loaded before anything else
  • main.* files - Loaded after everything else
  • Everything else loads in alphabetical order based on file name

Otherwise unopinionated

Meteor is great at locating all your code

Webpack/Browserify support available through packages

Pub Subs

Publications & Subscriptions

Distributed Data Protocol (DDP)

REST for websockets

Example


// In /lib
Posts = new Meteor.Collection('posts')

// On the server
Meteor.publish('postsByAuthor', function(author) {
  return Posts.find({ flagged: false, author: author });
});

// On the client
Meteor.subscribe('postsByAuthor', 'bob-smith');

Posts.find();
// Client will only see Posts that have been flagged false
//   and Posts with the author bob-smith
